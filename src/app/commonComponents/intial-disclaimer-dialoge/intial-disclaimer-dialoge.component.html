Application needs authentication. You can enter whatever email address you want and password. This is done to show you a router guards functioanlity.

Application is created on the version 16. Used interfaces/classes/enum as a best practices.

  1. Create multiple modules in angular application.
      There are 3 modules:
    • Custom common module: This module is used create common custom pipe for the value transformation like sorting of employee/checking value is null/ To clip the name/ Truncate the string/ some custom component imported like error handling
    • Employee Management system module: This module consist the CRUD operations i.e.. to create/ edit or update/ delete and view the list of employee. Used FormsModule, FontAwesomeModule, MatButtonModule, MatCardModule, NgSelectModule, MatTooltipModule, CustomPipesAndFunctions, MatDatepickerModule, MatIconModule, ReactiveFormsModule, MatRippleModule. User can also see the list of employee's on clicking on single employee you can see there brief details. This module have search by name functioanlity as well.
    • Team Reports module: This module is used to read only functioanlity. User can see the reports of progress of task team wise(Note: Progress are random dummy data for teams). User can see team wise reports then employee of particular teams and details fo particular employee (Three level drilling.)
  2. Create multiple components and link them using the routing concept.
      There are total 13 components used in this application.

      • Shared Components

      1. Error/Success Component: Component used for showing the error/success msg to user while adding/ updating the employee
      2. Confirmation Component: Component used for asking the consent from user before deleting the employee details
      3. Application header: Component used for showing the header of application which will be visible to user after the successfull authentication
      4. Application footer: Component used for showing the footer of application.
      5. Disclaimer component: Used to show all the Disclaimer of the application

      • Authentication

      1. Login Component: Component used for authentication the user

      • Employee Management module:

      1. Four Components: Component used for adding/ updating/ viewing employee and there details

      • Teams Reporting module:

      1. Three Components: Component used for viewing Teams/ employees in a teams and there details
  3. Share data among components using services
      There are 3 Services in this application.
    • Authentication: Service used to perform authentication of application like login/ logout functions
    • SetAndGetEmplyee : Service used to perform adding and updating the list of employee i.e sharing the data of employee between components
    • Utility Service : Service used to set the token to cookies and after succesfull authentication and remoing it when logout is perform
  4. Use Pipes
      There are 7-8 piped used in this application.
    • NullValuePipe, TruncatePipe, EmployeeSorting, ConfirmationDialogeComponent, GenderPipe, ErrorDialogeComponent, DesignationPipe, TeamPipe, ClipNamePipe, RolePipe there all pipes used
  5. JSON File not used
      Only created some constant for dropdown functioanlity and enums as App Constants
    • Designation list
    • Roles list
    • Teams list
    • Gender for radio
  6. Used dependency injection and validation at their respective places
    • Created Authguards using authentication dependency
    • Used created services as a dependency injection on respective places
    • Added validation in add/update emplyee places
  7. For Icons
    • Used material icons and material
  8. Last but not least also used guards
